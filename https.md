### 再说说 HTTP

- 建立 HTTP 连接的两台计算器可以当做是两个角色 一个是客户端,负责发起请求，一个是 response,负责给对应的请求作出回应
- HTTP 属于无状态协议，每条 HTTP 请求之间是没有任何相关性的，每一次请求都是独立的。
- 客户端请求类型：GET / POST / PUT / DELETE / HEAD / OPTIONS / TRACE / CONNECT
- 服务器的回应存在状态码

  1. 5 开头表示服务器错误
  2. 4 开头表示客户端错误
  3. 3 开头表示要进行进一步处理
  4. 2 开头表示成功。
  5. 1 开头表示在请求被接受处理的同时提供的额外信息

- 无论是 request 还是 response 双方都拥有一块头部信息。（称之为 Header）
  Header 中有固定个的关键字，当然也可以自定义头部字段，其用途是用于传递额外的信息。

### 对称加密 VS 非对称加密 (HTTPS中的)

- 加解密都使用同一种算法的加密手法，称之为对称加密，反之则为非对称加密

- 对称加密只需要一把钥匙

- 非对称加密 => 私钥 & 公钥

- 非对称加密算法需要两把钥匙，公钥和私钥，它们是一对。用公钥加密的密文只能用相应的私钥解开，用私钥加密的密文只能用相应的公钥解开。其中，公钥是公开的，私钥是不对外公开的。

- SSL 结合了这两种加密算法的有点。利用非对称加密算法来协商生成对称加密的密钥，然后之后就用对称加密来进行通信。
